中断

什么是中断

  • 操作系统引入了中断机制，实现了计算机可以多道程序并发执行
  • 发生了中断就代表着需要操作系统介入

如何中断

:::warning 假设想要进程1切换到进程2 :::

  • CPU收到计时部件发出的 中断信号 切换为核心态对中断进行处理
  • 操作系统内核就会对中断进行处理
  • 内核就会切换进程1从而执行进程2
  • 当中断发生的时候 当前进程会停止运行 等待操作系统对中断进行处理

用户态发出中断的系统调用 CPU会立即切换到内核态执行切换

用户态和核心态之间的切换

  1. 用户态到核心态中间的切换是通过中断来实现的 并且中断是唯一途径 调用int陷入指令会进入中断
  2. 核心态到用户态的切换通过执行一个特权指令 将程序状态字（psw）标志位设置为用户态
  1. 如果有中断发生 CPU会保留当前被中断的程序的CPU环境如“程序状态字”“程序计数器”“各种通用寄存器”
  2. 恢复的时候会恢复原进程的运行环境 并退出中断 返回原进程并继续往下面执行

中断的分类

  1. 内中断（指令中断，硬件故障，软件中断）（来源于CPU内部）
  2. 外中断（外部设备引起的中断）比如io设备发出的中断

系统调用

为什么需要系统调用

用户进程想要使用类似于打印机这种共享资源 只能通过系统调用向系统发出请求 操作系统会对各个请求进行协调管理

可以保证系统的安全性和稳定性 防止用户进行非法操作

系统调用的分类

  1. 设备管理 （完成设备的 请求 释放 启动 功能）
  2. 文件管理 （完成文件创作的 读写删除创建功能）
  3. 进程控制（完成进程的 创建 撤销 阻塞 唤醒等功能）
  4. 进程通信 （完成进程间的消息传递、信号传递等功能）
  5. 内存管理 (完成内存的分配/回收等功能)

0x00 syscall 基础概念

Windows下有两种处理器访问模式：用户模式（user mode）和内核模式（kernel mode）。用户模式下运行应用程序时，Windows 会为该程序创建一个新进程，提供一个私有虚拟地址空间和一个私有句柄表，因为私有，一个应用程序无法修改另一个应用程序的私有虚拟地址空间的数据；内核模式下，所有运行的代码都共享一个虚拟地址空间， 因此内核中驱动程序可能还会因为写入错误的地址空间导致其他驱动程序甚至系统出现错误。 内核中包含了大部分操作系统的内部数据结构，所以用户模式下的应用程序在访问这些数据结构或调用内部Windows例程以执行特权操作的时候，必须先从用户模式切换到内核模式，这里就涉及到系统调用。 x86 windows 使用 sysenter 实现系统调用。 x64 windows 使用 **syscall ** 实现系统调用。

进程

进程实体

进程实体是由 金城所需要的资源组成

为了方便操作系统管理 完成程序间的并发之心 所以引入了 进程实体的概念

进程实体由一下三个部分组成

  • pcb （他是进程存在的唯一标识）
  • 程序段 （存放程序代码）
  • 数据段 （存放程序运行需要的数据）比如变量

所谓的创建进程其实就是创建一个pcb 撤销进程也就是撤销pcb

进程的状态

  • 创建态 （正在创建 操作系统正在为进程分配资源 初始化pcb）
  • 运行态 （占有CPU并且在CPU上运行） （单核处理器 一个时刻只能有一个进程处于运行态）
  • 就绪态 （已经具备运行条件 但是没有空闲CPU所以不能运行）
  • 阻塞态 （应等待某一个事物 所以暂时不能运行）
  • 终止态 （进程正在从系统中撤销 系统会回收进程拥有的资源撤销pcb）

进程控制

进程控制是对操作系统中的进程 进行有效管理 它具有创建进程 撤销进程 进程的状态转换等功能

如何实现进程状态转换

修改pcb信息 修改相应的队列

原语

为了防止 进程状态转换 这修改pcb内容 而没有修改 相对应的队列 所以采用用原语实现进程控制

原语的特点是执行期间不允许中断 只能一气呵成 这种不可被中断的操作就是原语操作

通过 关中断 && 开中断 实现原子操作

  • 关中断之后 收到系统中断 不会被中断 要等待原子操作完成
  • 原子操作完成之后 使用开中断 这个时候可以被中断

进程通信

为了系统安全 一个进程是不允许访问另外一个进程的内存空间的
进程通信就是进程间的信息交换

共享空间

两个进程之间不能互相访问 所以操作系统创建了一个 两个进程都能够访问的共享空间 共享空间的访问是互斥的 同一时间只能有一个进程操作 共享空间的两种方式

进程通信-共享存储

  1. 基于数据结构的共享 （低级）
  2. 基于存储区的共享 （高级） 在内存中划分出一块区域用于共享

进程通信-管道通信

管道的本质是一个特殊的共享区域 其实就是在内存当中开辟一个大小固定的缓冲区 管道都是半双工的 管道的访问也是互斥的

线程概念-多线程模型

线程是一个基本的CPU执行单位，也是程序执行流中的最小单位 每个线程都有个tcb和进程的pcb相同 线程分为用户级线程和内核级线程

  • 用户级线程通过程序管理 包括线程切换 用户级线程内核是看不到的
  • 内核级线程 通过系统内核管理 执行状态转换都需要在内核态进行

内核级线程才是CPU的处理单位
